# 🍸 ALECHEMY — Презентация проекта

## Умный помощник для планирования вечеринок с алкоголем

---

## 📌 Общее описание

**Alechemy** — это полнофункциональное веб-приложение для планирования мероприятий с алкоголем. Приложение помогает пользователям организовывать вечеринки, рассчитывать количество напитков и еды, отслеживать потребление алкоголя, находить заведения поблизости, а также взаимодействовать с друзьями через социальные функции и геймификацию.

**Целевая аудитория:** люди от 18 лет, которые планируют вечеринки, хотят контролировать своё потребление алкоголя или ищут рекомендации по коктейлям и заведениям.

---

## 🎯 Ключевые функции

### 1. 📅 События и сценарии
- Создание событий с выбором готового сценария (День рождения, Корпоратив, Свидание и т.д.)
- Настройка количества гостей, напитков, блюд и продолжительности
- Автоматические рекомендации по подготовке, проведению и восстановлению после события
- Приглашение друзей на события

### 2. 🧮 Калькулятор алкоголя (BAC)
- Расчёт приблизительного уровня алкоголя в крови (Blood Alcohol Content)
- Учитывает вес, пол и количество выпитого
- Помогает планировать безопасное употребление

### 3. 📊 Алко-дневник
- Журналирование употреблённых напитков
- История потребления за дни/недели/месяцы
- Аналитика и графики

### 4. 🗺 Карта заведений
- Интерактивная карта баров, магазинов, ресторанов
- Информация об акциях и скидках
- Фильтрация по типу заведения и локации

### 5. 🍹 База коктейлей
- 100+ коктейлей с детальными рецептами
- Категоризация (классические, шоты, тики, сауэры и т.д.)
- Рейтинги и отзывы пользователей
- Поиск по ингредиентам

### 6. 🛒 Список покупок (Premium)
- Автоматический расчёт продуктов для мероприятия
- Основывается на количестве гостей, продолжительности и интенсивности
- Ингредиенты для выбранных коктейлей и блюд

### 7. 🏆 Геймификация
- **Баллы** — за активность (создание событий, отзывы, приглашение друзей)
- **Достижения (Ачивки)** — за выполнение определённых условий
- **Челленджи** — еженедельные и специальные задания
- **Таблица лидеров** — соревнование с друзьями и глобально

### 8. 👥 Социальные функции
- Система друзей с поиском по уникальному тегу (#ABC123) или email
- Приглашения на события
- Совместное редактирование событий
- Публичные профили

## 📦 Структура модулей

### accounts/ — Пользователи и авторизация
- **Функции:** регистрация, логин, профиль, друзья, уведомления
- **Модели:** `User` (кастомный), `Profile`, `FriendRequest`, `Notification`
- **API:** JWT аутентификация, CRUD профиля
- **Особенности:** 
  - Уникальные теги для поиска друзей (#ABC123)
  - Подтверждение возраста 18+
  - GDPR consent
  - Premium статус

### events/ — События и сценарии
- **Функции:** создание событий, выбор сценария, алко-дневник
- **Модели:** `Event`, `Scenario`, `Drink`, `Dish`, `DrinkCategory`, `AlcoholLog`
- **Особенности:**
  - 8 категорий сценариев (Свидание, Вечеринка, Корпоратив и т.д.)
  - Три этапа события (подготовка, во время, восстановление)
  - Автоматический расчёт напитков на основе гостей и продолжительности

### gamification/ — Геймификация
- **Функции:** баллы, достижения, челленджи, лидерборд
- **Модели:** `UserScore`, `Achievement`, `UserAchievement`, `Challenge`, `UserChallenge`
- **Особенности:**
  - Автоматическое начисление баллов за действия
  - Прогресс-бары для челленджей
  - Глобальный и друзья-лидерборд

### recipes/ — База коктейлей
- **Функции:** список коктейлей, рецепты, отзывы, поиск
- **Модели:** `Cocktail`, `CocktailIngredient`, `CocktailReview`
- **Категории:** Классический, Шот, Тики, Сауэр, Физ, Хайбол, Мартини, Замороженный, Горячий, Пунш, Шприц

### shopping/ — Список покупок
- **Функции:** генерация списка на основе события, ручное редактирование
- **Модели:** `ShoppingList`, `ShoppingItem`, `ScenarioSupplyTemplate`
- **Формула:** `количество = qty_per_person_per_hour × люди × часы × коэффициент_интенсивности`

### places/ — Карта заведений
- **Функции:** карта, список заведений, акции
- **Модели:** `Place`, `Promotion`
- **Типы:** Бар, Магазин, Другое

### stats/ — Статистика
- **Функции:** дашборд, графики потребления, PDF отчёты
- **Интеграция:** Chart.js для визуализации

### social/ — Социальные функции
- **Функции:** поиск друзей, запросы в друзья, рейтинг
- Модели вынесены в accounts/ для избежания циклических зависимостей

## 🔐 Безопасность

### Защита на уровне приложения

| Механизм | Реализация |
|----------|-----------|
| **Хэширование паролей** | Argon2 (победитель PHC) |
| **CSRF защита** | Django CSRF middleware + SameSite cookies |
| **XSS защита** | Content Security Policy (CSP) middleware |
| **SQL Injection** | Django ORM (параметризованные запросы) |
| **Rate Limiting** | Кастомный middleware + Redis |
| **Brute Force** | Ограничение попыток логина |
| **2FA** | TOTP для админ-панели (django-otp) |
| **CAPTCHA** | hCaptcha на регистрации/логине |
| **Session Security** | HttpOnly, Secure, SameSite cookies |

### Production Security Headers

```python
# HTTPS/SSL
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True

# HSTS
SECURE_HSTS_SECONDS = 31536000  # 1 год
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True

# Content Security
SECURE_CONTENT_TYPE_NOSNIFF = True
SECURE_BROWSER_XSS_FILTER = True
X_FRAME_OPTIONS = 'DENY'
```

### Content Security Policy (CSP)

Динамический CSP с nonce для inline scripts:
```
default-src 'self';
script-src 'self' 'nonce-{random}' https://js.hcaptcha.com;
style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
img-src 'self' data: https:;
font-src 'self' https://fonts.gstatic.com;
```

## 🔌 REST API

### Аутентификация

Все API endpoints используют JWT токены (Access + Refresh).

```bash
# Получение токена
POST /api/v1/auth/token/
{
  "email": "user@example.com",
  "password": "secret"
}

# Ответ
{
  "access": "eyJ0eXAiOiJKV1QiLCJhbGc...",   # 15 мин
  "refresh": "eyJ0eXAiOiJKV1QiLCJhbGc..."  # 7 дней
}

# Использование
Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGc...
```

### Основные Endpoints

| Endpoint | Метод | Описание | Auth |
|----------|-------|----------|:----:|
| **Аутентификация** ||||
| `/api/v1/auth/token/` | POST | Получить JWT токен | ❌ |
| `/api/v1/auth/token/refresh/` | POST | Обновить access токен | ❌ |
| `/api/v1/auth/register/` | POST | Регистрация | ❌ |
| `/api/v1/auth/me/` | GET | Данные пользователя | ✅ |
| `/api/v1/auth/password/change/` | POST | Смена пароля | ✅ |
| `/api/v1/auth/logout/` | POST | Logout (blacklist token) | ✅ |
| **События** ||||
| `/events/api/events/search/` | GET | Поиск событий | ✅ |
| `/events/events/recommendations-preview/` | POST | Рекомендации | ✅ |

## 📈 Обоснование технических решений

### Почему Django?

1. **Быстрая разработка** — встроенные компоненты (ORM, Auth, Admin)
2. **Безопасность** — CSRF, XSS, SQL injection защита из коробки
3. **Масштабируемость** — проверено на крупных проектах (Instagram, Pinterest)
4. **Экосистема** — тысячи готовых пакетов (DRF, django-otp, etc.)
5. **Документация** — одна из лучших в индустрии

### Почему PostgreSQL?

1. **Надёжность** — ACID транзакции, репликация
2. **Функциональность** — JSON, Full-text search, GIS
3. **Производительность** — оптимизатор запросов, индексы
4. **Масштабирование** — read replicas, partitioning

### Почему Argon2?

1. **Безопасность** — победитель Password Hashing Competition
2. **Защита от GPU атак** — memory-hard алгоритм
3. **Гибкость** — настройка time/memory cost
4. **Рекомендации** — OWASP рекомендует как #1 выбор

### Почему Docker?

1. **Консистентность** — одинаковое окружение dev/staging/prod
2. **Изоляция** — каждый сервис в своём контейнере
3. **Простота деплоя** — `docker-compose up` и готово
4. **Масштабирование** — легко добавить воркеры

### Почему Server-Side Rendering (Django Templates)?

1. **SEO** — страницы индексируются поисковыми системами
2. **Производительность** — меньше JavaScript, быстрее First Paint
3. **Простота** — один стек, один язык (Python)
4. **Безопасность** — меньшая поверхность атаки (нет отдельного API для SPA)

*Для будущего мобильного приложения уже есть REST API на DRF.*
